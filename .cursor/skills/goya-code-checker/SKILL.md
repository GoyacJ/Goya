# Goya 代码规范检查和修复

## 描述
检查代码是否符合 Goya 规范，并提供修复建议。

## 使用场景
- 代码评审
- 规范检查
- 自动修复
- 重构建议

## 检查项

### 1. 命名规范
- [ ] 类名大驼峰
- [ ] 方法名小驼峰
- [ ] 常量全大写
- [ ] 包名全小写
- [ ] 无下划线（数据库字段除外）

### 2. 注解使用
- [ ] 必须使用 @Slf4j
- [ ] Service 使用 @RequiredArgsConstructor
- [ ] 事务指定 rollbackFor
- [ ] 禁止 @Autowired 字段注入

### 3. 注释规范
- [ ] 类注释完整
- [ ] 方法注释完整
- [ ] 使用中文注释
- [ ] 包含 @author 和 @since

### 4. 日志规范
- [ ] 使用 log 而非 System.out
- [ ] 日志格式正确
- [ ] 日志级别合理

### 5. 异常处理
- [ ] 不吞掉异常
- [ ] 使用业务异常
- [ ] 记录错误日志

### 6. 安全规范
- [ ] 密码加密
- [ ] SQL 防注入
- [ ] XSS 防护
- [ ] 权限验证

## 使用方法

只需说："检查这段代码的规范性"

## 输出格式

```
【规范检查报告】

❌ 发现 3 处问题：

1. 命名不规范
   位置：UserService.java:15
   问题：方法名使用下划线 get_user_by_id
   建议：改为 getUserById

2. 缺少注解
   位置：UserService.java:10
   问题：缺少 @Slf4j 注解
   建议：添加 @Slf4j

3. 注释缺失
   位置：UserService.java:15
   问题：方法缺少注释
   建议：添加方法注释

【自动修复】
已生成修复后的代码
```
